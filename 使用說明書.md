## P3 — 任務編碼系統（時技影複）

**建議排版：** 左右對照表 + 計算公式範例

### 四維度定義表

| 維度 | 全名 | L（低）= 0.25 點 | M（中）= 1.0 點 | H（高）= 2.0 點 |
|------|------|-----------------|----------------|----------------|
| 時 | 耗時 | < 1 天 | 1-2 天 | > 2 天 |
| 技 | 技術 | 例行工具操作 | 需專業技能 | 高度專業（如 AI 研究） |
| 影 | 影響 | 1-5 人 | 6-20 人 | > 20 人或跨部門 |
| 複 | 複雜 | 單向溝通 | 多方協作 | 立場衝突需協調 |

### 點數計算公式

```
單任務點數 = 時值 + 技值 + 影值 + 複值

範例：編碼 3211 → 時=H(2.0) + 技=M(1.0) + 影=L(0.25) + 複=L(0.25) = 3.50 點

行動點數（圖表 Y 軸）= 所有任務點數總和 ÷ 4
```

**圖說策略：** 上方為 2×4 表格（四維度 × 三等級），下方為實際編碼拆解圖（每個 digit 對應一個維度，顏色區塊標示 L/M/H）。

**建議排版：** 決策樹 + 評分表

### 狀態判定流程（決策樹）

```
                ┌────────────────┐
                │ 本週任務數=0?  │
                │ 或點數=0?      │
                └────────┬───────┘
                  Yes ↓       ↓ No
             ┌──────────┐  ┌──────────────────┐
             │前後週正常? │  │點數>平均×1.25   │
             │           │  │且任務>平均×1.15? │
             └────┬──────┘  └────┬─────────────┘
               Yes↓   ↓No   Yes↓        ↓No
             [休假] [不存在] [豐盛]  ┌────────────┐
                                    │點數>平均×0.8│
                                    └──────┬─────┘
                                       Yes↓   ↓No
                                     [正常] ┌──────────┐
                                            │點數>平均×0.5│
                                            └─────┬─────┘
                                             Yes↓    ↓No
                                          [緊急]   [危險]──→另判斷是否回升中
```

### 綜合評分（A/B/C/D）

加權公式：

| 項目 | 權重 | 滿分條件 |
|------|------|---------|
| 點數 vs 個人平均 | 30% | ≥ 1.2 倍 → 100 分 |
| 任務數 vs 個人平均 | 15% | ≥ 1.2 倍 → 100 分 |
| 狀態等級 | 25% | 豐盛=98, 正常=95, 休假=75, 緊急=60, 危險=20~35 |
| 效率（每任務點數 vs 平均） | 15% | ≥ 1.2 倍 → 100 分 |
| 趨勢（較上週） | 15% | 上升 >10% → 90 分 |

```
A 級(≥80) = 優 / B 級(≥60) = 良 / C 級(≥40) = 可 / D 級(<40) = 待加強
```

**圖說策略：** 左半頁為決策樹（Yes/No 菱形 + 圓角矩形狀態），右半頁為加權評分表格 + 色條。狀態旁的 icon（🌟豐盛/🟢正常/🔴緊急/🟡危險/🔷休假）要保留。

## 這個系統解決的問題 & 提供的思路

### 解決的問題

1. **工作無法量化的困境** → 用四維度編碼（時技影複）將任務質量化為分數，每週每人有客觀的產出指標
2. **策略目標與日常脫鉤** → 每筆任務自動分類到 6 大策略，讓策略不再只是牆上的口號，而是每週的實際行動
3. **主管無法掌握部屬狀態** → 狀態公式（豐盛/正常/緊急/危險/休假）+ 綜合評分 A~D，一目瞭然
4. **溝通成本高、遺漏多** → 每週檢討建議從數據出發（四維度箭頭、演化指標），減少憑印象判斷
5. **部署複雜** → 雙擊 .bat 即完成 Notion 拉取 + 分類 + 圖表注入 + Netlify 上傳，非技術人員可操作

### 提供的思路

1. **編碼即評語**：用 4 個數字代替傳統的文字評等，比 5 分制更細、比自由描述更一致
2. **策略分類用關鍵字模型**：不追求完美精確，而是用關鍵字抓「概念與行為模型」— 夠用就好，調整也簡單
3. **狀態公式模擬主管直覺**：把主管多年對部屬的「感覺」歸納成可重現的數學規則（前後週正常+本週歸零→休假；高平均 1.25 倍→豐盛）
4. **演化指標捕捉長期變化**：廣度（是否在做不同的事）、深度（是否越做越難）、結構變化（策略是否轉移）、舒適區（是否在重複打轉）
5. **行動建議依角色分化**：同一筆數據，對「自己」和對「主管」產出不同的行動步驟，因為兩種角色的槓桿不同
